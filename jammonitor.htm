WAN Policy

Drag interfaces to change their bonding status. Primary accepts only one WAN.

Primary (1 max - others bond to this)
Bonded (aggregated with Primary)
Standby (failover only)

How WAN Priority Works:
Primary ‚Äî The main connection (only one allowed). All MPTCP traffic originates from this interface.
Bonded ‚Äî These interfaces work together with Primary simultaneously. Traffic is split across all of them for bandwidth aggregation. More bonded connections = more combined bandwidth.
Standby ‚Äî Failover connections that remain dormant until Primary and Bonded interfaces fail. When failover occurs, all Standby interfaces activate together.
Disabled ‚Äî Interface is completely turned off and not used.
Tip: Click on a WAN name to edit its settings.
